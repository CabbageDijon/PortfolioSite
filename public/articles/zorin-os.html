Migrating to Zorin OS on an Acer Laptop

Possibly one of the hardest projects I have undertaken. The Acer Aspire 3, specifically the A315-56-366R model, has been one many have had issues getting Linux to run on easily. After many attempts, different distros, and hours of different fixes from others who have either tried to install Linux in general on these laptops (The A315-56 range) or those who have magically succeeded. I had finally found a solution that worked for me.

Picking a Distro

In all brutal honesty, I went with Zorin OS because of my past usage of it on a family member's laptop. It worked perfectly on the Dell Inspiron laptop and was such a beautiful, smooth experience. Previous attempts used Mint and Ubuntu, getting to different stages which I do not feel were OS dependent but simply what I used when I tried a different fix. Also being Ubuntu based helped a lot due to my Pop!OS main system and the Zorin OS Dell.

The System in Question

Whilst not a high-end laptop, the Acer Aspire 3 A315-56-366R is a decent laptop for the price. It has an Intel i3-1005G1 CPU, 8GB of RAM in a 4GB soldered + 4GB SODIMM layout, and a 1TB 5400RPM HDD. Powered by an InsydeH20 BIOS, a key feature in our issues and install dificulties as you will soon read. The rest of the specs are not of much important but still include a 15.6" 768P TN display, Intel UHD G1 Graphics and the crunchiest looking webcam.

Before you carry on!

This is by no means an installation guide and not guranteed to work, but can very much be used to do the install process just note that I have only recently done this and long term stability (over normal usage and/or after updates) is fully unknown. Standard modern Linux kernels fail to boot or discover storage media on this motherboard configuration out-of-the-box. This failure stems from architectural flaws within the InsydeH2O UEFI ACPI tables combined with proprietary power management routing.

Preinstall Preparations

Before I could even get my laptop out I had to make sure I had the needed ISO from the official Zorin OS website. Once I had downloaded the latest version, I used a tool called balenaEtcher to flash the ISO onto a USB drive (I used a 64GB USB 3.0 drive, but any good drive over 8GB will surfice). Once the ISO was selected and set to flash to the USB, I left the process run then ejected the USB once complete. Now a lot of systems will allow you to just plug this USB into them when turned off then notice the drive and ask if you want to boot from it, this Acer is not as convenient due to a few key features mentions earlier. Without the USB in turn on the laptop and press F2/Delete immediately to enter the BIOS. It might ask you a password, if you know it enter it, if not then do 3 wrong attempts and a screen with Enter Unlock Password with show, select it using the arrow keys and enter. Once there a 10-digit code will show, simply use an online tool to decrypt this. Once you get in the BIOS, go to Main and press Ctrl+S to reveal the SATA Mode option in your menus. Select this and change it to AHCI if not already set to this. Now go to the Boot tab, select Secure Boot and set it to disabled. You can either navigate to the Save or similar tab, or press F10 to save and exit. Now you can go onto the next stage of booting from the USB.

Installing Zorin OS

Ensuring the device is off first, plug your Zorin OS prepared USB into the device, then turn it back on again. Once one, spam F12 till a screen appears asking which drive to boot from. Select your USB and press Enter, it will now boot into the USB. A GRUB menu with appear with a bunch of options, select Try or Install Zorin OS. Allow to load fully until a welcoming UI appears, click Try Zorin OS then leave it as it is. Using Ctrl+Alt+T ,or pressing your Windows key and searching Terminal, open the terminal as we need to run the following command to check storage devices deteched 

bash lsblk

It should show your USB and hopefully your main drive, usually as /dev/nvme or /dev/sda, depending on your storage device installed. If your internal storage doesn't show, no need for panic as this is the first and most frustrating issue many people come across which is luckily very easy to fix. Either close the lid, wait a bit and open it back up (can sometimes not fully work) or run the following command to make the system suspend to RAM and power down then wake back up 3 seconds later 

sudo rtcwake -m mem -s 3

Once the device is back on, you can rerun the bash lsblk command and your internal drive should now be visible. Go back to the home screen and launcher the Install Zorin app on screen, then follow the on-screen instructions to meet your requirements.


Recommendations

  • Select Install Third-party software for Graphics and WiFi hardware, this fetches all needed drivers to have you ready to go out the box
  • If using a low speed hard drive like the 5400RPM units commonly found on laptops, unchecking the download updates while installing

After following the on screen prompts throughout the install the system will require a reboot. Because a full restart cuts hardward power rails completely, you will trigger the cold-boot bug (Hard drive not visible issue) again on the first normal startup. There is no worry here since you will then be booted into the fallback terminal which look like a full black screen with some text in the top left. If you do not have any issues and boot right in, I envy you and your loving laptop. Terminal for those who get it, will show the below text or similar 

(initramfs) _ .

The Big Fix: Making it bootable

For some, this is the biggest roadblock they have encountered in their attempts, with the previous fix or the simple close and reopen laptop trick being commonly tried. Whilst for some this works for the boot issues, no one wants to have to do all of this each time they want to turn on their laptop. We will do a more permenant fix that applies itself on each boot for a seemless experience

Jumpstart the laptop

Very similar to what we did earlier, we will deliver a command to put the laptop to sleep but this time we need to manually wake it back up. Enter the below command. 

echo mem > /sys/power/state

If you have a power state LED on your laptop (LED that shows if your laptop is on, charging or asleep) wait for it to enter the sleep state then press the Spacebar. If your laptop has no power state LED, you can just wait 5 seconds then press the Spacebar.

It will power back up, for some the initramfs command line will appear but if you are like me you will get a fully blank screen. No need to panic, either way you get the next step is the same. Simply type in the following then hit the Enter key to execute it. 

exit

The system will immediately mount the needed filesystem and you will be launched into the graphical desktop environment.


Automating the process

To abstract this entire issue away from the end user, we deploy a low-level executable hook inside the local top-level assembly directories of the initramfs compilation suite. This enforces the S3 sleep cycle automatically before the kernel attempts to discover or mount root targets. All the following is to be run in the Terminal inside of Zorin OS. Open with either Ctrl+Alt+T or searching for it the launching. Since you are in the OS, I would recommend opening this page in a browser on the system so you can copy paste commands into the Terminal.

1. Generating the hook script target

sudo nano /etc/initramfs-tools/scripts/local-top/acer_sata_fix

2. Inject the following production shell script exactly as written 

#!/bin/sh
# /etc/initramfs-tools/scripts/local-top/acer_sata_fix

PREREQ=""

prereqs() {
    echo "$PREREQ"
}

case "$1" in
    prereqs)
        prereqs
        exit 0
        ;;
esac


. /scripts/functions

log_begin_msg "Executing Acer SATA controller ACPI wake workaround"


if [ -e /sys/class/rtc/rtc0/wakealarm ] && [ -e /sys/power/state ]; then
    

    echo 0 > /sys/class/rtc/rtc0/wakealarm
    
   
    if [ -e /sys/class/rtc/rtc0/since_epoch ]; then
        NOW=$(cat /sys/class/rtc/rtc0/since_epoch)
        ALARM=$((NOW + 3))
        
        
        echo "$ALARM" > /sys/class/rtc/rtc0/wakealarm
        
        
        
        echo mem > /sys/power/state
        
        
        sleep 2
        
        
        for host in /sys/class/scsi_host/host*/scan; do
            echo "- - -" > "$host"
        done
        
        log_success_msg "Acer SATA interface re-probed. HDD maps updated successfully."
    else
        log_failure_msg "System Error: Unable to read hardware RTC epoch."
    fi
else
    log_failure_msg "System Error: Critical sysfs power/RTC components unavailable."
fi

exit 0

Save and close the editor via Ctrl + O, Enter, and Ctrl + X.

For those who want to know the full happenings of the code they pasted, it goes as follows:

    Import canonical initramfs functional logging arrays
#
    # Verify systemic existence of target sysfs nodes for core PM and RTC
    # Clean out any pre-existing or orphaned hardware real-time alarms
    # Extrapolate current hardware clock timestamp and register a +3s wake window
    # Flash the alarm state into the hardware control register
    # Command the host hardware into an S3 Suspend state
    # The motherboard shuts down power lines, waking 3 seconds later via RTC interrupt
    # Post-wake optimization delay: give the spinning mechanical platter time to initialize
    # Execute a hard hardware rescan across every functional SCSI/SATA host channel

Authorize global system root ownership and elevate permissions to mark the hook file as an executable binary element: 

sudo chown root:root /etc/initramfs-tools/scripts/local-top/acer_sata_fix
        sudo chmod +x /etc/initramfs-tools/scripts/local-top/acer_sata_fix
       
sudo chmod +x /etc/initramfs-tools/scripts/local-top/acer_sata_fix

Force a system-wide rebuild of the primary initial ramdisk array across all compiled kernel versions currently indexed by the system: 

sudo update-initramfs -u -k all

Your system is should now be fully sorted, you can restart it to test the fixes implemented.
