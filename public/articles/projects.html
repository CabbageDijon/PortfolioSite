Projects by Tema

A running log of the projects I have built, what inspired each one, and the technical decisions behind them. This is where I share the story behind the code.

Every project starts with a problem. Sometimes it is a real-world issue — like needing weather data for F1 race weekends. Other times it is a personal itch — like wanting a better way to track IT support tickets. Either way, each project teaches me something new about web development, and I wanted a place to document that journey.

RaceWeather

This was the first project that went live and actually got used by other people. It pulls real-time weather data from an API and displays it on a dashboard tailored for F1 race weekends — track temperature, wind speed, rain probability, all updated automatically.

The technical stack is vanilla JavaScript with API integration. No frameworks, no build tools — just a clean, fast page that loads instantly. It taught me a lot about API rate limits, caching strategies, and designing a layout that works on both desktop and mobile while showing dense data.

RepoTicket

An IT ticket report system built with React, TypeScript, and PocketBase. This is currently in development, but the core idea is a self-hosted tool for tracking and resolving support requests — a lightweight alternative to heavy enterprise ticketing platforms.

The frontend uses React with Vite, Tailwind CSS, and React Query for server state. PocketBase handles the backend, auth, and database in a single binary. It is a great combo for solo projects that need real-time data without maintaining a full Node backend.

This Portfolio

The site you are reading this on is itself a project. Built with vanilla HTML, CSS, and JavaScript — no frameworks, no build step. It is deployed with Docker and Nginx, which keeps it fast and cheap to host.

I designed it to be a clean, editorial-style layout that feels more like a personal magazine than a typical portfolio site. The journal section, the expandable tools in Studio, and the contact form backend — each part is a small project in its own right.

What is Next

I have a few more tools planned for the Studio section — a QR code generator, an APR calculator, and a Sudoku solver. Each one is a focused, single-purpose utility that should be quick to build and actually useful. I will document the development process here as I go.
